## 1. 작업 개요

본 보고서는 **2026-04-20** 기준으로 수행된 다음 작업을 한곳에 묶어 기록한다.

1. **PRD 작성 시 참고 자료 매핑**  
   - PRD 본문 골격은 **Report/4**(Epic·여정·스토리·Gherkin)를 축으로 하고, **Report/1**(문제·동기), **Report/2**(기능·계약·불변조건), **Report/3** 및 **`.cursor/rules`**(품질·개발 원칙)에서 가져오는 구성을 권장·확정했다.

2. **제품 요구사항 문서(PRD) 신규 작성**  
   - 저장소 루트에 **`docs/`** 디렉터리를 두고, 그 안에 **`PRD_MagicSquare_4x4_TDD.md`**를 생성했다.  
   - 구현 코드는 추가하지 않았고, 기존 합의 범위를 넘는 **새 기능 요구사항은 넣지 않았다**.

3. **PRD 초안 이후 정밀 보정**  
   - FR-04 승인 기준(AC-FR-04-02 등)을 **자동 검증 가능한 문장**으로 다듬었다.  
   - Traceability 표에서 I1 행의 **테스트·컴포넌트** 열을 Boundary와 Domain 검증 경로가 함께 드러나도록 보강했다.

## 3. 근거 문서와의 정렬

| 근거 | PRD에서의 역할 |
|------|----------------|
| **Report/1** | 문제를 “불변조건·계약으로 닫는다”는 관점, 입출력 명확성 필요성 |
| **Report/2** | I1~I6, 입출력 스키마, Domain/Boundary 오류 코드·메시지 표, 컴포넌트 명·Dual-Track 테스트 ID 참조 |
| **Report/3**·**`.cursorrules`**·**`.cursor/rules/*.mdc`** | 커버리지 목표, ECB·TDD·금지 패턴 등 비기능·공정 원칙 |
| **Report/4** | Epic·페르소나·여정·User Story·Gherkin·TQ-01 예시 행렬 |

**범위 결정:** Report/2에 기술된 Data 레이어(저장소)는 PRD **Out-of-Scope**로 두어, 사용자가 요청한 **UI/DB/Web 없이 순수 로직 중심** 범위와 맞췄다.

## 4. PRD가 만족시킨 제약(체크리스트)

| 제약 | 이행 |
|------|------|
| 구현 코드 미작성 | `docs/`에 Markdown 요구사항만 존재 |
| 요구사항 범위 유지 | N×N 일반화·생성기·UI/DB 등은 Out-of-Scope로 명시 |
| 모호어 금지 | “적절히” 등 없이 수치·문자열·판정으로 고정 |
| 테스트 가능 문장 | 각 FR에 AC-ID 부여, TC-ID·E2E-ID와 연계 |
| Dual-Track 반영 | §8에 Boundary 트랙 / Domain 트랙 / 병렬 RED·GREEN·REFACTOR |
| Concept → … 추적성 | §12 Traceability Matrix 필수 열 구성 |
| 입출력 계약 고정 | 사용자가 제시한 Input/Output/Magic constant와 Report/2 표와 정합 |

## 5. 테스트 데이터 및 검증 의무

PRD **§9.3**에 다음을 명시했다.

- **TQ-01:** Report/4 Gherkin과 동일한 4×4 예시(정상 경로).  
- **TQ-NOSOL:** 해 없음 시나리오용 후보 행렬.  
- **검증 의무:** 구현 착수 전 자동 테스트로 `TQ-NOSOL`에 대해 FR-05 두 시도가 모두 실패함을 확인하고, 그렇지 않으면 PRD의 표를 수정한다.

---

## 6. 선행·후행 문서 관계(권장)

현재 `Report/4` 개요 표는 Report 1~3만 “선행 문서”로 열거한다. **PRD가 요구사항 베이스라인으로 추가**되었으므로, 향후 Report 4를 갱신할 때 **“PRD: `docs/PRD_MagicSquare_4x4_TDD.md`”**를 참조 링크로 넣으면 Epic→PRD→구현 추적이 한 줄로 이어진다(본 보고서 작성 시점에는 Report 4 본문은 변경하지 않음).

---

## 7. 향후 작업(선택)

- `TQ-NOSOL`에 대한 **자동 검증 테스트** 추가 후, PRD §9.3 “검증 의무”를 **완료**로 표시  
- 구현 진행 시 PRD §12 Traceability와 **pytest 커버리지 리포트**를 주기적으로 대조  
- 필요 시 **Report 4** Level 1 표에 PRD 링크 한 줄 추가
