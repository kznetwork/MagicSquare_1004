## 6. Dual-Track RED — 테스트 ID 및 계약

### 6.1 입력·출력·상수(프로젝트 조건 요약)

| 구분 | 내용 |
|------|------|
| 입력 | `int[][]` 4×4, `0`은 빈칸, 빈칸 **정확히 2개**, 나머지 값 **1~16**, 0 제외 **중복 없음** |
| 출력 | `int[6]` = `[r1, c1, n1, r2, c2, n2]`, 좌표 **1-index** |
| 마방진 상수 | 4×4 → 행·열·두 대각선 합 **34** |

**초기 합의(설계 단계):** 표만 있을 때는 구현·GREEN·REFACTOR 없이 **계약·ID**만 정의했다. 이후 §7에서 **pytest 스켈레톤**과 **LOG-RED-01 단일 GREEN**이 추가되었다(나머지 ID는 여전히 RED 또는 미구현).

### 6.2 TRACK A — UI / Boundary RED

| Test ID | Scenario (요약) | Invariant 보호 내용 |
|---------|-------------------|---------------------|
| **UI-RED-01** | 4×4가 아닌 `int[][]`로 진입점 호출 시 예외 | 그리드는 반드시 4행·4열 |
| **UI-RED-02** | 빈칸(0) 개수가 2가 아니면 예외 | “정확히 두 빈칸” 계약 |
| **UI-RED-03** | 1~16 밖 값이 있으면 예외 | 0=빈칸 전용, 나머지 1~16 |
| **UI-RED-04** | 0 제외 숫자 중복 시 예외 | 1~16 부분집합 무중복 |
| **UI-RED-05** | 솔루션 반환은 길이 **6** | `[r1,c1,n1,r2,c2,n2]` 스키마 |
| **UI-RED-06** | 반환 좌표 `r*`,`c*`는 **1~4**(1-index) | 출력 좌표 체계 |

### 6.3 TRACK B — Logic RED

| Test ID | Scenario (요약) | Invariant 보호 내용 |
|---------|-------------------|---------------------|
| **LOG-RED-01** | `find_blank_coords`: 빈칸 **2개**, **row-major** 순서 | 첫/둘째 빈칸의 전역 순서 |
| **LOG-RED-02** | `find_not_exist_nums`: 누락 **2개**, **오름차순** | 채울 숫자 집합·표현 순서 |
| **LOG-RED-03a** | `is_magic_square`: 행만 34, 열/대각 깨짐 → `False` | 행만으로는 불충분 |
| **LOG-RED-03b** | 열만 맞고 행 깨짐 → `False` | 행·열 대등 판정 |
| **LOG-RED-03c** | 행·열 34이나 대각선 하나라도 ≠34 → `False` | 두 대각선 포함 정의 |
| **LOG-RED-03d** | 알려진 완성 마방진 → `True`, 합 **34** | `MAGIC_SUM`과 일치 |
| **LOG-RED-04a** | `solution`: 반환 길이 6, 순서 `[r1,c1,n1,r2,c2,n2]` | 출력 스키마 |
| **LOG-RED-04b** | `solution`: 모든 좌표 1-index·범위 1~4 | UI 계약과 정합 |
| **LOG-RED-04c** | `solution`: small→첫 빈칸 실패 시 **reverse** 시도 정책이 결과에 반영 | 탐색/백트랙 계약 |
| **LOG-RED-04d** | `solution`의 `(n1,n2)`가 `find_not_exist_nums`와 **집합 일치** | 해와 누락 수 일관성 |

### 6.4 진행 상태 메모

- **UI-RED-01~06:** `pytest.fail("RED")` 스켈레톤만 존재 → 실행 시 실패(RED).  
- **LOG-RED-02~04d:** 동일.  
- **LOG-RED-01:** `find_blank_coords` 최소 구현으로 **통과(GREEN)**. 다음 GREEN은 보통 `find_not_exist_nums` 또는 UI 경계 검증 등 한 ID씩 진행한다.
