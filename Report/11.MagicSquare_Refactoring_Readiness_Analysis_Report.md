## 3. ECB(Entity–Control–Boundary) 분석 요약

| 파일 | ECB 역할 요약 |
|------|----------------|
| `domain.py` | **상태를 들인 Entity 클래스는 없고**, `list[list[int]]`에 대한 **순수 도메인 연산(규칙 실행)**이 중심이다. 사용자 구분(데이터 보관 vs 규칙 실행)으로는 **Control(비즈니스 규칙·연산 실행)**에 가깝다. |
| `boundary.py` | 외부 입력에 `ValueError`·고정 메시지로 검증하고 도메인을 호출한다. **UI와 도메인 사이의 응용 경계(검증·오케스트레이션·예외 매핑)**로 Boundary 역할에 부합한다. |
| `gui/main_window.py` | 위젯·이벤트·표현이 주이며 도메인 규칙을 풀어 쓰지 않는다. **이중 `validate` 호출**은 레이어 혼입이라기보다 **중복 호출**에 가깝다. |

**잘못된 레이어 배치:** 명백한 위반은 없다(도메인에 Qt 없음, GUI에 마방진 판정 없음).

**`domain.py` 내부 Entity vs Control이 논의될 만한 경계:** `solve_placement`는 빈 칸·빠진 수 조회 후 두 배치를 시도하는 **유스케이스 조율(Control 성격)**에 가깝고, `find_blank_coords`, `find_missing_pair_sorted`, `is_completed_magic_square`, `_copy_grid`는 **그리드에 대한 규칙·질의**에 가깝다. 클래스 단위 ECB를 엄격히 적용할 경우 **`solve_placement`를 먼저 분리하기 쉬운 경계**로 볼 수 있다.

## 7. Dual-Track REFACTOR 실행 기록 (커밋 `ffb680c`)

본 절은 **단일 커밋**으로 수행한 Dual-Track 준수 리팩터·테스트 보강을 기록한다. **공개 계약**(입력·출력 `int[6]` 1-index, FR-05 두 시도 순서, `ValueError` 메시지 문자열, 예외 타입)은 **변경하지 않았다.**

### 7.1 기준선(Step 0)

- `python -m pytest -q` 실행 시, **Logic/UI RED 스켈레톤 14건**이 `pytest.fail("RED")`로 실패하여 **전체 GREEN이 아니었음**을 확인했다.
- 동일 커밋에서 **계약 테스트 보강(Step 2)** 후 **Logic R-L2 리팩터(Step 3)**를 적용하고, **23 passed**로 GREEN을 확인했다.

### 7.2 선택한 리팩토링 목표

| ID | 트랙 | 내용 |
|----|------|------|
| **(선행)** | Logic + Boundary 테스트 | RED 제거·PRD §9.3 TQ-01 등 **기존 동작 고정 assertion** (신규 기능 아님) |
| **R-L2** | Logic (`domain.py`) | 행·열·주·부 대각 **합 계산**을 `_sum_row`, `_sum_col`, `_sum_main_diagonal`, `_sum_anti_diagonal`로 추출. `is_completed_magic_square`는 0 검사 후 동일 순서로 위 함수들을 호출하도록만 정리(매직 넘버 추가 없음). |

### 7.3 테스트 보강 요약

**Logic — `tests/magicsquare/test_magic_square_logic_red.py`**

- TQ-01 고정 행렬 `_TQ_01` 유지.
- `find_missing_pair_sorted(_TQ_01) == (1, 6)`.
- `is_completed_magic_square` 반례: 행만 `MAGIC_SUM`인 `_GRID_ROWS_ONLY`, 그 전치(열 조건만 만족하는 형태), 행·열은 `MAGIC_SUM`이나 대각은 아닌 `_GRID_SEMI_NOT_MAGIC`.
- 완성 격자: TQ-01 빈칸에 `(6, 1)` 채운 `_tq01_completed_magic()`에 대해 `True`.
- `solve_placement(_TQ_01) == [3, 3, 6, 4, 4, 1]`, 좌표 1~4, `{n1,n2}={1,6}`, **작은 수 먼저 배치 실패·큰 수 먼저 성공**을 단계별로 검증.

**Boundary(UI 트랙 테스트) — `tests/magicsquare/test_magic_square_ui_boundary_red.py`**

- `validate`: 0 개수 ≠ 2, 값 범위 위반, 비영 0 중복 시 `str(exc) == MSG_*` **전문 동일 비교** (`pytest.raises(..., match=)`는 메시지에 괄호가 있어 정규식 특수문자와 충돌하므로 사용하지 않음).
- `solve(_TQ_01)`: 길이 6, **`[3, 3, 6, 4, 4, 1]`**과 동일(도메인 모듈 import 없이 Boundary 계약만 검증).
