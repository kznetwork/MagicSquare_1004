## 1. 작업 개요

다음 작업이 수행되었다.

1. **공통 상수 (`constants.py`)**  
   - `MATRIX_SIZE`에 더해 `CELL_MIN_VALUE`, `CELL_MAX_VALUE`(= `MATRIX_SIZE * MATRIX_SIZE`), `MAGIC_SUM`(34), PRD §8.1과 **문자 단위 동일**한 실패 메시지 상수(`MSG_*`)를 정의했다.

2. **Domain (`domain.py`) — Track B 확장**  
   - `find_blank_coords`: 기존과 동일 계약(행 우선·1-based 두 좌표).  
   - `find_missing_pair_sorted`: 1~16 중 격자에 없는 두 수를 오름차순 `(n_small, n_large)`로 반환.  
   - `is_completed_magic_square`: 0이 없을 때 4행·4열·주·부 대각 **10선**이 모두 `MAGIC_SUM`과 일치하는지 판정.  
   - `solve_placement`: FR-05에 따라 첫 빈칸·둘째 빈칸에 `(n_small, n_large)` 시도 후 실패 시 `(n_large, n_small)` 시도; 성공 시 `[r1,c1,n1,r2,c2,n2]` 반환, 모두 실패 시 `NoMagicSolutionError`.  
   - 입력 격자는 **비변형**: 시도용 격자만 `_copy_grid`로 복사해 사용.

3. **Boundary (`boundary.py`) — Track A 확장**  
   - `validate`: BR-10 순서 — I1(크기·`None` 행) → I2(값 범위) → I3(0의 개수 2) → I4(1~16 중복). 실패 시 `ValueError`와 **§8.1 고정 `message`**.  
   - `solve`: `validate` 후 `solve_placement` 호출; `NoMagicSolutionError` → `ValueError(MSG_NO_SOLUTION)`로 매핑.

4. **PyQt6 Screen (`magicsquare/gui/`)**  
   - `gui/app.py`: `QApplication` + `MainWindow` 표시·이벤트 루프.  
   - `gui/main_window.py`: `MATRIX_SIZE`×`MATRIX_SIZE` `QSpinBox`(0~`CELL_MAX_VALUE`), **「풀기」** 클릭 시 `validate` → `solve` 순 호출, 실패 시 `QMessageBox.warning`, 성공 시 결과 라벨에 쉼표 구분 표시.  
   - **Qt import는 `gui/` 패키지에만** 두었으며, `domain.py`·`boundary.py`에는 PyQt 의존성이 없다.

5. **실행 진입점 (`__main__.py`)**  
   - `python -m magicsquare` → `gui.app.run_app()`.  
   - `PyQt6` 미설치 시 안내 메시지와 함께 종료(`pip install -e ".[gui]"`).

6. **패키지 메타 (`pyproject.toml`)**  
   - `[project.optional-dependencies].gui = ["PyQt6>=6.5"]` 추가.

7. **온보딩 문서 (`README.md`)**  
   - **그래픽 UI (PyQt6, 선택)** 소절: `pip install -e ".[gui]"` 및 `python -m magicsquare` 한 줄 실행 경로 문서화.

8. **Git 커밋(로컬 기준 예시)**  
   - `122c4db` — `feat(magicsquare): domain blank coords and boundary grid shape validate` (상수·`domain`·`boundary` 초기·테스트 import 정리 등 선행 작업).  
   - `d4ac2e2` — `feat(gui): PyQt6 main window and domain solve via boundary validate` (본 보고서의 GUI·해 탐색·README·`pyproject` 반영).

## 6. 알려진 사항·문헌 정합

1. **「풀기」핸들러**에서 `validate` 후 `solve`를 호출하며, `solve` 내부에서도 `validate`를 다시 호출하므로 **검증이 이중**이다. 동작에는 문제 없으며, 후속에서 UI는 `solve`만 호출하도록 단순화할 수 있다.  
2. **PRD AC-FR-02-01**의 TQ-01 첫 빈칸 좌표 `(3,4)` 표기와 §9.3 행렬 리터럴(행 3에서 0이 **3열**) 사이에 불일치가 있을 수 있다. 구현·기존 테스트는 **행렬 리터럴** 기준 **(3,3), (4,4)**를 사용한다.  
3. **U-RED-02~06·LOG-RED-02 이후**는 아직 RED 스켈레톤일 수 있으므로, 본 구현과 **문구·예외 타입**을 맞추려면 해당 테스트를 순차 GREEN으로 전환하는 작업이 남는다.
